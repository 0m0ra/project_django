# To-Do List — Список задач

Простое и удобное веб-приложение для управления личными задачами, созданное на Django. Приложение позволяет пользователям создавать, организовывать и отслеживать свои задачи с помощью интуитивного интерфейса, календарного представления и системы напоминаний.

To-Do List — это полнофункциональное приложение для управления задачами, которое помогает пользователям организовать свою работу и ничего не забыть. Приложение предоставляет современный дизайн, плавные анимации, систему аутентификации для персональных списков задач, календарное представление с привязкой задач к датам, а также раздел обратной связи для связи с администратором.

## Features

- ✅ **Управление задачами** — создание, редактирование, удаление и отметка выполнения задач
- ✅ **Календарное представление** — визуализация задач по датам с навигацией по месяцам
- ✅ **Сроки выполнения** — привязка задач к конкретным датам с визуальной индикацией просроченных
- ✅ **Система аутентификации** — регистрация, вход и выход пользователей с персональными списками задач
- ✅ **Обратная связь** — форма для отправки сообщений администратору
- ✅ **Статистика** — отображение общего количества задач, выполненных и активных
- ✅ **AJAX-взаимодействие** — плавное обновление интерфейса без перезагрузки страницы
- ✅ **Адаптивный дизайн** — корректная работа на всех устройствах (десктоп, планшет, мобильный)
- ✅ **Админ-панель** — полное управление задачами и сообщениями через Django Admin
- ✅ **Валидация данных** — многоуровневая проверка данных на клиенте и сервере

## Installation

### Предварительные требования

- Python 3.8 или выше
- pip (менеджер пакетов Python)
- Git (для клонирования репозитория)

### Пошаговая инструкция

1. **Клонирование репозитория**

   ```bash
   git clone <url-репозитория>
   cd project
   ```

2. **Создание виртуального окружения**

   ```bash
   # Windows
   python -m venv venv
   venv\Scripts\activate

   # Linux/Mac
   python3 -m venv venv
   source venv/bin/activate
   ```

3. **Установка зависимостей**

   ```bash
   # Windows (рекомендуется)
   python -m pip install -r requirements.txt
   
   # Linux/Mac (или если pip добавлен в PATH)
   pip install -r requirements.txt
   ```

4. **Применение миграций**

   ```bash
   python manage.py makemigrations
   python manage.py migrate
   ```

5. **Создание суперпользователя**

   ```bash
   python manage.py createsuperuser
   ```

   Введите имя пользователя, email и пароль при запросе.

6. **Сбор статических файлов** (опционально, для production)

   ```bash
   python manage.py collectstatic --noinput
   ```

7. **Запуск сервера разработки**

   ```bash
   python manage.py runserver
   ```

8. **Открытие в браузере**

   - Главная страница: http://127.0.0.1:8000/
   - Календарь: http://127.0.0.1:8000/calendar/
   - Контакты: http://127.0.0.1:8000/contact/
   - Регистрация: http://127.0.0.1:8000/register/
   - Вход: http://127.0.0.1:8000/login/
   - Админ-панель: http://127.0.0.1:8000/admin/



## ER-диаграмма базы данных

![ER-диаграмма]

### Описание структуры

```
┌─────────────────┐
│      User       │
│  (Django Auth)  │
├─────────────────┤
│ id (PK)         │
│ username        │
│ email           │
│ password        │
│ ...             │
└────────┬────────┘
         │
         │ 1:N
         │
┌────────▼────────┐      ┌──────────────────┐
│      Task       │      │ ContactMessage   │
├─────────────────┤      ├──────────────────┤
│ id (PK)         │      │ id (PK)          │
│ title           │      │ name             │
│ completed       │      │ email            │
│ created_at      │      │ message          │
│ updated_at      │      │ created_at       │
│ due_date        │      │ is_read          │
│ user_id (FK)    │──┐   └──────────────────┘
└─────────────────┘  │
                     │
                     │ (опциональная связь)
                     │
                     └─── User может иметь много Task
```

### Связи

- **User** (1) ──< (N) **Task**: Один пользователь может иметь множество задач
- Связь опциональная: задачи могут существовать без пользователя (для демо-режима)
- **ContactMessage**: Независимая модель для хранения сообщений обратной связи

### Основные таблицы

- `auth_user` — пользователи системы (Django Auth)
- `todo_app_task` — задачи пользователей
- `todo_app_contactmessage` — сообщения обратной связи

### Поток данных

1. **Браузер** → отправляет HTTP-запросы через AJAX или обычные формы
2. **Django URL Router** → маршрутизирует запросы к соответствующим views
3. **Views** → обрабатывают бизнес-логику, валидируют данные через формы
4. **Models (ORM)** → взаимодействуют с базой данных
5. **SQLite Database** → хранит все данные приложения
6. **Django Admin** → предоставляет интерфейс для управления данными
